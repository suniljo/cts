Spring Data JPA uses specific annotations to map Java objects (entities) to relational database tables and define the relationships between them. The four primary relationship types are: 

1. @OneToOne: A single instance of one entity relates to exactly one instance of another entity (e.g., a user and a user profile).

2. @OneToMany: A single instance of one entity relates to multiple instances of another entity (e.g., a department and its employees).

3. @ManyToOne: Multiple instances of one entity relate to a single instance of another entity. This is often the inverse side of a @OneToMany relationship and typically owns the foreign key.

4. @ManyToMany: Multiple instances of one entity relate to multiple instances of another entity (e.g., a student and multiple courses)

Unidirectional vs. Bidirectional
================================
A relationship defined using Spring Data JPA can be either unidirectional, where only one of the related entities is aware that the relationship exists, or bidirectional, where both entities are aware of the relationship.

What does "aware" mean? In short, an entity that is aware of a relationship has a field that references the related entity

Directionality:
1. Unidirectional: Only one side of the relationship is aware of the other.
2. Bidirectional: Both entities have references to each other, allowing navigation in both directions.

In a unidirectional one-to-one relationship, only one of the entities is aware of and can access the other entity in the relationship. This contrasts with a bidirectional one-to-one relationship, where both entities involved are aware of each other and can mutually access each other's data.


Owning Side: The "owner" of the relationship is the entity responsible for managing the foreign key in the database.
In bi-directional relationships, the side without the "mappedBy" attribute is the owner.

Owning Side vs. Inverse Side: In any bidirectional relationship, one side is the "owning side" and manages the foreign key(s) in the database. The other side is the "inverse side" and uses the "mappedBy" attribute to indicate it does not own the relationship. JPA only considers the owning side when saving or updating relationships.

In JPA entity relationships, there is always an "owner" of the relationship. Figuring out the owner of the relationship is different for uni and bi-directional relationships.

In uni-directional relationships, the owning side is the side that is aware. Whereas in bidirectional relationships, the entity that does not have a mappedBy annotation parameter is the owning side.


mappedBy: Used on the non-owning side of a bi-directional relationship to tell JPA that the relationship is managed by the other side.

Cascading: Defines which operations (like saving or deleting) should "cascade" from a parent entity to its related children. Commonly used types include PERSIST, MERGE, and REMOVE




Fetch Types: Control how related entities are loaded from the database.
1. FetchType.LAZY: The related data is only loaded when explicitly accessed (default for @OneToMany and @ManyToMany).
2. FetchType.EAGER: The related data is loaded immediately along with the primary entity (default for @OneToOne and @ManyToOne).

package com.example.demo.entity;

import jakarta.persistence.*;
import com.fasterxml.jackson.annotation.JsonManagedReference;

@Entity
@Data
public class Person {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long personId;

    private String personName;
    private String personGender;

    @OneToOne(mappedBy = "person", cascade = CascadeType.ALL)
    @JsonManagedReference
    private Passport passport;

    // Getters and Setters

    public Long getPersonId() {
        return personId;
    }

    public void setPersonId(Long personId) {
        this.personId = personId;
    }

    public String getPersonName() {
        return personName;
    }

    public void setPersonName(String personName) {
        this.personName = personName;
    }

    public String getPersonGender() {
        return personGender;
    }

    public void setPersonGender(String personGender) {
        this.personGender = personGender;
    }

    public Passport getPassport() {
        return passport;
    }

    public void setPassport(Passport passport) {
        this.passport = passport;
        passport.setPerson(this); // maintain bi-directional mapping
    }
}

package com.example.demo.entity;

import jakarta.persistence.*;
import com.fasterxml.jackson.annotation.JsonBackReference;
import java.time.LocalDate;

@Entity
public class Passport {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long passportId;

    private String passportNumber;
    private LocalDate issuedDate;
    private LocalDate expirayDate;

    @OneToOne
    @JoinColumn(name = "person_id")
    @JsonBackReference
    private Person person;

    // Getters and Setters

    public Long getPassportId() {
        return passportId;
    }

    public void setPassportId(Long passportId) {
        this.passportId = passportId;
    }

    public String getPassportNumber() {
        return passportNumber;
    }

    public void setPassportNumber(String passportNumber) {
        this.passportNumber = passportNumber;
    }

    public LocalDate getIssuedDate() {
        return issuedDate;
    }

    public void setIssuedDate(LocalDate issuedDate) {
        this.issuedDate = issuedDate;
    }

    public LocalDate getExpirayDate() {
        return expirayDate;
    }

    public void setExpirayDate(LocalDate expirayDate) {
        this.expirayDate = expirayDate;
    }

    public Person getPerson() {
        return person;
    }

    public void setPerson(Person person) {
        this.person = person;
    }
}

Mapper Utility (Manual Mapping)

package com.example.demo.mapper;

import com.example.demo.dto.PersonDTO;
import com.example.demo.dto.PassportDTO;
import com.example.demo.entity.Person;
import com.example.demo.entity.Passport;

public class PersonMapper {

    // Entity → DTO
    public static PersonDTO toDTO(Person person) {
        if (person == null) return null;

        PersonDTO dto = new PersonDTO();
        dto.setPersonId(person.getPersonId());
        dto.setPersonName(person.getPersonName());
        dto.setPersonGender(person.getPersonGender());

        if (person.getPassport() != null) {
            PassportDTO passportDTO = new PassportDTO();
            passportDTO.setPassportId(person.getPassport().getPassportId());
            passportDTO.setPassportNumber(person.getPassport().getPassportNumber());
            passportDTO.setIssuedDate(person.getPassport().getIssuedDate());
            passportDTO.setExpirayDate(person.getPassport().getExpirayDate());

            dto.setPassport(passportDTO);
        }

        return dto;
    }

    // DTO → Entity
    public static Person toEntity(PersonDTO dto) {
        if (dto == null) return null;

        Person person = new Person();
        person.setPersonId(dto.getPersonId());
        person.setPersonName(dto.getPersonName());
        person.setPersonGender(dto.getPersonGender());

        if (dto.getPassport() != null) {
            Passport passport = new Passport();
            passport.setPassportId(dto.getPassport().getPassportId());
            passport.setPassportNumber(dto.getPassport().getPassportNumber());
            passport.setIssuedDate(dto.getPassport().getIssuedDate());
            passport.setExpirayDate(dto.getPassport().getExpirayDate());

            // maintain bi-directional mapping
            passport.setPerson(person);
            person.setPassport(passport);
        }

        return person;
    }
}


Service Layer

package com.example.demo.service;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import com.example.demo.dto.PersonDTO;
import com.example.demo.entity.Person;
import com.example.demo.mapper.PersonMapper;
import com.example.demo.repository.PersonRepository;

import java.util.List;
import java.util.stream.Collectors;

@Service
public class PersonService {

    @Autowired
    private PersonRepository personRepository;

    public PersonDTO savePerson(PersonDTO dto) {
        Person person = PersonMapper.toEntity(dto);
        Person saved = personRepository.save(person);
        return PersonMapper.toDTO(saved);
    }

    public List<PersonDTO> getAllPersons() {
        return personRepository.findAll()
                .stream()
                .map(PersonMapper::toDTO)
                .collect(Collectors.toList());
    }

    public PersonDTO getPersonById(Long id) {
        return personRepository.findById(id)
                .map(PersonMapper::toDTO)
                .orElse(null);
    }

    public void deletePerson(Long id) {
        personRepository.deleteById(id);
    }
}

Create Mapper Interface 

package com.example.demo.mapper;

import org.mapstruct.*;
import com.example.demo.dto.PersonDTO;
import com.example.demo.dto.PassportDTO;
import com.example.demo.entity.Person;
import com.example.demo.entity.Passport;

@Mapper(componentModel = "spring")
public interface PersonMapper {

    // Entity → DTO
    PersonDTO toDTO(Person person);

    PassportDTO toPassportDTO(Passport passport);

    // DTO → Entity
    Person toEntity(PersonDTO dto);

    Passport toPassportEntity(PassportDTO dto);

    // Handle bi-directional mapping manually after mapping
    @AfterMapping
    default void linkEntities(@MappingTarget Person person) {
        if (person.getPassport() != null) {
            person.getPassport().setPerson(person);
        }
    }
}
